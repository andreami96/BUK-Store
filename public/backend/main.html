Specification

Purpose


The BUK Store application was developed during the Hypermedia Application (Web and Multimedia) course of the Politecnico di Milano. The main purpose of this application is to develop a web application for an online book store and its multimedia content about books and authors. In particular the web application should handle 3 main entities: Books, Authors and Events. Each book can be written by different authors, and of course each author has probably written more than one book. There are also events during which a book is presented.
So the user should be able to surf the BUK Store application, finding the useful information about each book, event or author, but also he or she should be able to reserve some books and to see his/hers purchases.


Web Architecture


The BUK Store Web Application is divided into the common Client-Server pattern. The client is represented by the user's web browser, and for that the frontend part of the application is written with HTML, CSS and JavaScript, in this way the most common browsers can handle and render properly the frontend of this application.
Focusing on the Server, it is developed using the famous Node.js framework and it is deployed on a Heroku machine

Other Information


Our teams is composed by 3 members, and we have decide to split, in general, the amount of work into two main parts: Frontend development and Backend development.
First of all, in order to properly split our work, we have chosen to devote some times to think and design the REST API of our application all together, in order to have a common reference point. Then, before starting to work on both the frontend and the backend, we also have decided to firstly define the user experience of the BUK Store application, spending some times in designing the interfaces
At this point we was ready to start, and since, thanks to Node.js, the Server development is become easier during the years, we have decide to initially split our group in this way:

  • 1 person is in charge of the backend development: Luca Napoletano
  • 2 people are in charge of the frontend development: Andrea Miotto & Claudio Montantari

With this division in mind, we have started to work both on the frontend and on the backend, and this was possible thanks to development of the API before starting to coding.
As soon as the backend development was finished, all the team members started to work on the frontend speeding up the implementation of the UI/UX.
