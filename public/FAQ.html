This page is the available documentation for the BUK Store Backend application. Here you can find the main information concerning the development of the Backend and the development of the Database used inside the application.
This document is structured into 4 more section:

  1. Team
  2. Links
  3. Specification
  4. Other Information

In the first section you will find the main information about MiChi-Studios team and how they split their work in order to be able to complete this project. Moreover here you can find a contact address for each team member

In the second section are reported the links related to each required deliverable, concerning the Backend part of this project.
Indeed in this section you will find the links for the download of the YAML specification file, the link to the SwaggerUI online page in order to test the BUK Store API, a link used to download our application in a convenient ZIP file and finally you will find the link of the MiChi-Studios GitHub Repository

In the third section of this document you will find a detailed report of each part of our application. Here there is a section related to our web architecture, and here you can find how we have decided to divide our application and how each component interacts with the others.
Then there is a part devoted to the explanation of the Database of our application, so how we have developed our relationship inside a real database, what are the entities we have used and how these entities are mapped into the OpenAPI models
This section continues with a part on the tools we have used during the development phase of the BUK Store application, what kind of languages and frameworks we have adopted in our web app.
Finally this part ends with a discussion on our decision, and how the choices we have made adhere to the OpenAPI specification we have written, how we have partitioned the application data and the static assets and how we have managed the session of the user inside the web application

In the last section of this documentation page you will find a more detailed report concerning how our team decided to split the work of building this application and what each team member has learned during this experience
